# **9. Capstone Project Milestones**

You will build one project throughout the roadmap: a Blog API. Each phase adds features. By week 12 it will be a production-grade REST API you can show in your portfolio.

## **Blog API — Feature Progression**

- Week 3: Basic CRUD for users (in-memory), no database yet

- Week 5: Plug in PostgreSQL + TypeORM — persist users and posts

- Week 6: Add validation, pagination, filtering on all endpoints

- Week 7: Add auth — register, login, JWT tokens, protected routes

- Week 8: Add roles — admin can delete any post, users can only delete their own

- Week 9: Add Swagger docs, file upload for post thumbnails, config module

- Week 10: Add Redis caching on GET /posts, rate limiting on auth routes

- Week 11: Write unit + integration tests — target 80%+ coverage

- Week 12: Postman E2E test suite, Dockerize, deploy to Railway

> **Portfolio Tip**
> After week 12, your Blog API README should include: architecture diagram, API documentation (Swagger link), how to run locally (Docker Compose), test instructions, and a live URL. This is what hiring managers look for.
